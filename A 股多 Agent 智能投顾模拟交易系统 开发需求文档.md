# A 股多 Agent 智能投顾模拟交易系统 开发需求文档

## 项目整体定位

基于 Python + LangChain + FastAPI + Vue3 + SQLite 开发**A 股多智能体量化投顾系统**。

核心逻辑架构：

1. **AI 策略选股模块**为**公共底层基础能力**；

2. 既独立对外提供 Web 对话窗口，给用户自然语言提问、策略选股、持仓诊断建议；

3. 同时内嵌作为工具，被所有模拟交易子 Agent 自动调用选股、自主决策交易；

4. 模拟炒股是独立子 Agent 生态，全自动闭环运行，复用底层选股能力但不受人工实时干涉。

   仅做A 股主板（包括中小板），禁用创业板、科创板、北交所；严格遵循 A 股 T+1、涨跌停、交易费率实盘规则，支持多子 Agent 模拟交易、分批挂单、正 T 反 T、自动复盘、日志知识库、策略回测、Vue 全功能可视化前端。

## 一、技术栈（强制固定）

- 后端语言：Python 3.10+
- Agent 框架：LangChain
- Web 后端：FastAPI
- 前端框架：Vue3 + Vite + ECharts（禁止 React、禁止原生 HTML）
- 数据库：SQLite 本地文件存储
- 数据处理：pandas 处理 baostock K 线、均线、财报数据
- 日志复盘：本地结构化 Log 文件 + 标准 Markdown 复盘文档

## 二、数据源与数据规则

### 1. 数据源

- 股票 / 大盘日线 K 线、指数：baostock 官方接口， 建议把csv数据保存到本地， 如果缺了增量走baostock更新即可， 否则容易被限流
- 基本面：baostock 个股季报财报数据
- 个股业务 & 所属板块：预留搜索接口，结果**本地 SQLite 缓存**，避免重复调用
- 分钟级 K 线：暂不实现，预留架构扩展接口，当前只做**日线**（用前复权）， 大盘指数用上证指数就好了

> 当前stock_analysis_xulu项目似乎搞好下载了一些csv数据， 以及编写好了代码， 还有一些策略选股的代码， 你需要在这个项目重构， 或者你参考这个项目新起一个项目， 随你安排

### 2. 均线规则

- 固定自动计算 4 条均线：**5 日、10 日、20 日、60 日均线**
- 保留后续新增均线的配置入口，但当前仅启用这 4 条

### 3. 板块逻辑

不接入实时新闻资讯（避免消息面滞后）；

通过个股主营业务推断所属板块，每日量化计算**板块热度分值**：

维度包含板块涨停家数、连板高度、成交量放量程度，供 Agent 选股决策优先选用高热度主板板块。

## 三、交易硬约束规则（不可改动）

1. 标的限制：只允许 60/00 开头 A 股主板，强制屏蔽创业板、科创板、北交所

2. 交易制度：严格遵守**T+1**，支持**正 T、反 T**所有合规做 T 方式

3. 涨跌停约束：一字涨停 / 跌停当日禁止买卖；普通涨跌停按正常价格逻辑判定

4. 交易费率（每笔）

   - 券商佣金：万 0.854 双向收取
   - 印花税：万 5 仅卖出单向收取

   

5. 初始本金：每个独立 Agent 默认初始本金 **150000 元**

6. 条件单机制：不限制固定数量，支持**列表式任意多条挂单**

   

   包含：分批买入单、分批卖出单、止损单、止盈单、价格条件触发单

7. 撮合规则：条件单价格落在当日最低价～最高价区间内，按设定价格成交

## 四、核心功能模块

### 模块 1：公共 AI 策略选股模块（底层公共能力）

1. 内置 Python 实现标准化策略

   - 龙头打板战法
   - 动量趋势策略
   - 20/60 日均线回调企稳策略

   

2. 支持用户**自然语言自定义策略**，AI 自动解析规则并筛选股票

3. 输入：日线 K 线、四大均线、大盘指数、个股季报、板块热度

4. 输出：候选股票列表、匹配理由、关键技术指标

5. 双重用途

   - 独立给 Web 前端 AI 对话窗口使用，服务用户手动选股、持仓问诊
   - 作为内部工具，自动被模拟交易子 Agent 调用选股

   

### 模块 2：策略仓库功能（新增优化需求）

1. 支持策略保存、命名、编辑、删除、分类管理
2. 所有子 Agent 可从策略仓库一键绑定对应战法策略
3. 内置策略 + 用户自定义策略统一纳入仓库管理

### 模块 3：多子 Agent 模拟交易系统

1. Agent 管理采用**工厂模式**，不写死数量，支持动态新增、删除、重命名、配置子 Agent

2. 初始预置 2 个默认 Agent

   - Agent1：固定追高打板龙头战法策略， 追龙头涨停强势股票， 关注k线外关注涨停表现， 不要关注量比， 因为如果出现一字涨停成交量会很小， 你应该关注妖股中后期的换手率有没有打上来， 这样才能有资金接力， 看好
   - Agent2：完全自主决策 Agent，无人工提示词干涉，自行定策略、选股、交易

   

3. 支持导入用户真实持仓成本、数量，纳入系统模拟统计与盈亏计算

4. 新增 Agent 风控配置（优化需求）

   - 单只个股最大仓位占比限制
   - 单日最大亏损阈值，触发后停止新开仓
   - 全局整体仓位上限可配置，行情弱势可强制半仓 / 空仓

   

5. 每日全自动流程

   

   1）收盘更新最新日线 K 线、均线、持仓市值

   

   2）Agent 调用工具 + 底层选股能力，分析大盘 / 板块 / 个股

   

   3）生成多条次日条件单：分批买卖、止损、止盈全部写入数据库

   

   4）次日定时任务刷新行情，自动判定条件单是否触发、撮合成交

   

   5）自动更新 Agent 余额、持仓、仓位、交易记录

### 模块 4：日志 & 复盘 & 知识库系统

1. 两类日志独立存储、按日期 + AgentID 规范目录命名

   - thinking.log：记录 Agent 每一步决策思路、工具调用过程、参数、推理链路
   - 每日自动生成标准化 Markdown 复盘文档：当日收益、净值变动、操作总结、策略有效性复盘

   

2. 所有日志、复盘文档本地归档，作为 Agent 私有长期知识库，弥补 LLM 无状态缺陷

3. 前端支持按日期、Agent 名称、股票代码**关键词检索日志与复盘文档**

### 模块 5：历史策略回测模块

1. 回测数据源：历史日线 K 线 + 四大均线

2. 回测时间区间：内置快捷区间 + 自定义区间

   

   快捷区间：近 3 天、近 1 周、近 1 月、近一季度、今年以来

3. 支持对内置策略、用户自定义策略、单个 Agent 历史战绩做回测

4. 输出回测指标：年化收益、最大回撤、胜率、净值曲线、盈亏统计

### 模块 6：Vue3 可视化前端（全功能）

1. 总览大盘看板

   

   - 全 Agent 绩效卡片：本金、净值、总收益、跑赢大盘比例
   - 大盘指数走势、板块热度排行
   - Agent 收益率**策略排行榜**，按收益、回撤、跑赢大盘排序

   

2. 单个 Agent 详情页

   

   - 持仓明细、仓位占比、盈亏统计
   - 净值曲线并对标大盘走势
   - 个股 K 线图，标注 Agent 买卖点、止损止盈点位
   - 历史交易记录、当日待触发条件单列表
   - 在线查看每日 Markdown 复盘、thinking 日志

   

3. AI 对话选股页面

   

   - 自然语言输入策略，AI 推荐股票并给出逻辑
   - 支持导入个人持仓，AI 做持仓诊断、调仓建议

   

4. 策略回测页面

   

   - 选择策略、选择回测区间
   - 可视化回测曲线、量化指标报告

## 附加 agent接入
llm用deepseek v4pro， 官网文档
```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get('DEEPSEEK_API_KEY'),
    base_url="https://api.deepseek.com")

response = client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=[
        {"role": "system", "content": "You are a helpful assistant"},
        {"role": "user", "content": "Hello"},
    ],
    stream=False,
    reasoning_effort="high",
    extra_body={"thinking": {"type": "enabled"}}
)

print(response.choices[0].message.content)
```
其中 key 通过环境变量 DEEPSEEK_API_KEY 配置，不应写入仓库。
